# User Personas

Persona templates provide defaults. They never change platform contracts, workflow order, or code paths.

## Template Catalog

| Persona | Primary Need | Template |
|---|---|---|
| TTO | Proactive licensing discovery | TTO Licensing Pipeline |
| Corporate Innovation | Technology scouting | Technology Scouting |
| Venture Builder | Venture creation from science | Venture Pipeline |
| VC | Technical due diligence | Deal Flow Intelligence |
| PE | Acquisition technology risk | Tech Due Diligence |

## Shared Workflow Contract

All personas use the same loop:

Subscribe -> Ingest -> Score -> Review -> Act

No persona receives custom endpoints, custom engines, or special-case pipeline logic.

## Template Scope

Templates may define:

1. Default streams
2. Default dimensions
3. Default pipeline stages
4. Default saved views (Stage 2 W1+)

Templates must not define:

1. Persona-specific routing
2. Persona-specific data model branches
3. Persona-specific security policy

## Success Pattern by Persona

1. Identify high-signal objects quickly.
2. Understand entity context and score trajectory.
3. Move items into action pipeline with minimal friction.
